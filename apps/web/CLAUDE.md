# apps/web

Vite + React frontend for `sync_board`. See [repo root CLAUDE.md](../../CLAUDE.md) for monorepo-wide rules.

## Tech Stack

- Vite + React + TypeScript
- MUI component library with `@emotion` styling
- TanStack Query for server state (cards, columns, board data fetched from the backend)
- Zustand for client/UI state (selection, drag state, local-only flags)
- TanStack Router for routing (file-based, routes generated from `src/routes/`)
- Zod for runtime validation at boundaries
- Vitest + React Testing Library for tests

## Commands

```bash
pnpm dev          # Vite dev server with HMR
pnpm build        # tsc -b && vite build (typecheck is part of build)
pnpm lint         # ESLint
pnpm preview      # Serve the production build locally
pnpm exec tsc -b --noEmit   # Typecheck only (no emit)
```

Tests use:

```bash

pnpm test <pattern>     # filter by filename/describe
pnpm test --watch       # watch mode
```

## Folder structure

```
src/
  features/        # one folder per feature (board, cards, columns, auth, sync, ...)
    <feature>/
      components/  # feature-local components
      hooks/       # feature-local hooks
      store/       # feature-local state
      lib/         # feature-local utilities 
      index.ts     # public interface (only re-exports what other code may use)
  components/      # general / design-system components shared across features
  api/             # TanStack Query hooks + fetchers
  hooks/           # generic shared hooks
  lib/             # framework-agnostic utilities
  store/           # app-wide state
  routes/          # TanStack Router file-based routes
    __root.tsx     # root layout
  theme/           # MUI theme + emotion setup
  main.tsx         # entry: createRouter + providers (QueryClient, Theme, RouterProvider)
```

## Naming

- **Components**: `PascalCase.tsx` (`CardList.tsx`, `BoardHeader.tsx`).
- **Routes** (`src/routes/`): kebab-case filenames matching the URL segment (`board-settings.tsx` → `/board-settings`). See [Routing](#routing) for routing conventions.

See the root [CLAUDE.md](../../CLAUDE.md) for cross-cutting naming rules (modules, tests, types).

## Architecture rules (enforced or expected)

1. **Server state vs UI state.** Anything that lives on the server goes through TanStack Query. Zustand is only for ephemeral UI state (drag, selection, modals). Do not duplicate server data into Zustand.

## Routing

File-based routes under `src/routes/` (route tree auto-generated by TanStack Router).

- **Filenames**: kebab-case matching the URL segment (`board-settings.tsx` → `/board-settings`). See [Naming](#naming).
- **Special files** (`__root.tsx`, `index.tsx`, `_layout.tsx`, `$param.tsx`, etc.): follow TanStack Router's [file-based routing docs](https://tanstack.com/router/latest/docs/framework/react/routing/file-based-routing).
- **Thin routes.** A route file wires route options (loader, `validateSearch`, etc.) and renders a component imported from the relevant feature (`src/features/<feature>/`). No business logic or large component bodies in the route file itself.
- **Data loading.** Use the route `loader` to prefetch via `queryClient.ensureQueryData(...)`. The component still uses `useQuery` and gets data instantly — avoids render-then-fetch waterfalls. Don't fetch only from the component.
- **Search params.** Any route that reads search params must declare a Zod schema in `validateSearch`. `useSearch` is then type-safe end-to-end.
- **File vs folder routes.** Default to a single file per route (`posts.tsx`). Promote to a folder (`posts/route.tsx` + `posts/$postId.tsx` + ...) as soon as the route gains children. Don't pre-emptively create folders for leaf routes.
- **Code splitting.** One file per route — no `.lazy.tsx` split. Rely on TanStack Router's automatic code splitting.

## Sync handling

TanStack Query cache is the source of truth for rendered data. Incoming WS messages invalidate or directly update query data (`queryClient.setQueryData` / `invalidateQueries`). Optimistic updates for user actions are expected — reconcile on the server's authoritative echo.

On unresolvable conflict (local state malformed, echo doesn't match expected shape, version skew, etc.), don't try to patch — invalidate the affected query and refetch from the server. Authoritative state always wins.

## Code Style

- Functional React only. Hooks for state and effects.
- Style with MUI's `sx` prop or `styled()` from `@mui/material/styles` (emotion under the hood).
- Keep components small. Lift logic into hooks or lib.
